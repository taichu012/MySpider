----------------------------
REQ0008：实现了bat脚本在win下自动将目录下的html生成index page（非JAVA代码）
REQ0009：实现了bat脚本在win下自动打包资源文件为一个rar（带时间戳）来备份（需要将命令行的rar.exe放在同目录）

REQ0010：实现了bat脚本用来运行爬车。考虑到3个bat各自需要一定时间，所以不放在同一个bat中运行；详见readme.txt

	因为win7的cmd无法运行utf-8的bat脚本；而win10的cmd可以；

【OPEN】目前程序默认是：“5线程，site重试1次，每次请求间隔1秒”，这些都可配置，但先不做。
【OPEN】webmagic：增加需求（解决原来的缺陷）“添加资源的逻辑层次依赖关系，及预期结果”
	缺陷1：link一股脑放入pagerequest中，多线程分布式取，其中link可能是多个种类，它们之间地位等价；
	缺陷2：因为link或link指向的期望page之间没有加入其实已经知道的先验的依赖关系，所以无法高效的对page/link按照种类type来决定如何操作，导致pipeline不加区分的只针对page全集
	设计1：增加page之间的逻辑层次依赖关系，并运行根据关系定义操作，比如A1，A2，A3是平级的，成功一个就不用再尝试下载另外两个page等等，对挖掘的逻辑表达大大增强；
	设计2：基于“page之间的逻辑层次以来关系及操作定义”，还可加上“预期模型”，它可以用来审计挖掘的结果，统计结果成为知识库，方便以后转向AI；
	设计3：有了1和2的设计，很容易将process和pipeline更细化到“某一个page”，而非page全集上，表达能力更加强大；
	考虑将以上想法作为issue提交给webmagic项目，有机会可以参与开发。
